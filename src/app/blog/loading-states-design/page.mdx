# Designing delightful loading states

<Badge variant="secondary">UX</Badge> <Badge>Motion</Badge>

La performance percepita conta. Skeletons, aggiornamenti ottimistici, e micro‑interazioni riducono l'ansia d'attesa e rendono l'esperienza più fluida.

<Callout variant="success" title="Obiettivo">
  Mostrare come alternare tra skeleton e aggiornamento ottimistico con un piccolo playground.
</Callout>

## Pattern

1. Skeleton: occupa lo spazio del contenuto e comunica progresso
2. Ottimistico: applica subito la modifica e riconcilia quando arriva la risposta
3. Motion: piccoli hint di stato (pulse, shimmer) aiutano il contesto

```tsx
// Estratto del playground
const save = async () => {
  setIsSaving(true)
  if (mode === "optimistic") setUsername(draft)
  await sleep(1200)
  if (mode === "skeleton") setUsername(draft)
  setIsSaving(false)
}
```
